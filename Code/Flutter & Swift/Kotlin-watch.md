# 안드로이드 워치에서 음성 데이터를 녹음하여 음악 인식 기술을 적용시켰던 기억
1. 블루투스 리시버
2. 와이파이

- 휴대폰에 연결되어있지 않을때는 워치 자체에서 와이파이를 사용하여 네트워킹 후 간편하게 코드 구현이 가능했음

- 휴대폰에 연결되어있을때는 와이파이가 자동 해제되고 블루투스 연결만 남아 워치 자체적으로 네트워킹 하려면 블루투스 연결을 통해 휴대폰을 사용한 네트워킹을 해야했음

- 네트워킹은 구현이 어렵지 않았으나 문제는 블루투스로 내가 녹음한 녹음데이터까지 넘겨야함

- 바이너리 데이터로 변환해서

- 녹음한 데이터를 블루투스로 넘기려면 Flutter만으로 한계


순수 Kotlin으로 구현하면 Bluetooth 데이터이동을 좀더 간결한 코드로 구현할 수 있었으나
음악 인식 처리를 Flutter코드로 구현했고 와이파이 사용시에도 Flutter로 구현을 해놓았기 때문에 
 순수 Kotlin으로 다시 앱을 구현하는것은 비효율적이라고 생각함
- 데이터만 Kotlin으로 옮겨다니면서 이동시키자 생각


- Kotlin을 사용해서 단계적으로 해결
1) 워치(Flutter)에서 음성 녹음
2) 바이너리 데이터로 변환 후 Kotlin으로 옮김
3) Kotlin에서 휴대폰 Kotlin으로 옮김
4) 휴대폰 Kotlin에서 Flutter로 옮긴 후 네트워킹
5) 휴대폰 Flutter에서 API 결과 데이터를 Kotlin으로 이동
6) 휴대폰 Kotlin에서 워치 Kotlin으로 데이터 이동
7) 워치 Kotlin에서 Flutter코드로 이동
8) 결과 데이터 Flutter로 출력

(와이파이로 데이터 받아올때와의 코드 비교)
